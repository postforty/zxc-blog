# 게시글 작성 시 SEO 체크리스트

## 📝 필수 항목 (자동 처리됨)

이 항목들은 시스템이 자동으로 처리하므로 신경 쓰지 않아도 됩니다.

- ✅ **메타데이터 생성**: 제목과 요약에서 자동 생성
- ✅ **Sitemap 업데이트**: 빌드/재검증 시 자동 추가
- ✅ **구조화된 데이터**: 게시글 정보에서 자동 생성
- ✅ **Canonical URL**: 게시글 ID로 자동 설정
- ✅ **Open Graph 기본 정보**: 제목, 설명 자동 설정

## ⚠️ 작성자가 신경 써야 할 항목

### 1. 제목 (Title) - 중요도: 🔴 높음

**SEO 영향**: 검색 결과 순위에 직접적인 영향

- [ ] **길이**: 50-60자 이내 (한글 기준 25-30자)
- [ ] **키워드 포함**: 주요 키워드를 제목 앞부분에 배치
- [ ] **명확성**: 클릭을 유도하는 명확한 제목
- [ ] **중복 방지**: 다른 게시글과 제목이 겹치지 않도록

```
❌ 나쁜 예: "오늘의 개발 이야기"
✅ 좋은 예: "Next.js 13 App Router로 블로그 성능 50% 개선하기"
```

### 2. 요약 (Summary/Description) - 중요도: 🔴 높음

**SEO 영향**: 검색 결과 설명문으로 표시됨

- [ ] **길이**: 120-160자 이내
- [ ] **키워드 포함**: 자연스럽게 주요 키워드 포함
- [ ] **행동 유도**: 클릭을 유도하는 문구
- [ ] **정확성**: 게시글 내용을 정확히 요약

```
❌ 나쁜 예: "개발에 대한 글입니다."
✅ 좋은 예: "Next.js 13의 App Router와 Server Components를 활용하여 블로그 초기 로딩 속도를 50% 개선한 실전 경험을 공유합니다."
```

### 3. 이미지 alt 속성 - 중요도: 🟡 중간

**SEO 영향**: 이미지 검색 최적화, 접근성

- [ ] **모든 이미지에 alt 텍스트 작성**
- [ ] **설명적**: 이미지 내용을 구체적으로 설명
- [ ] **키워드 포함**: 자연스럽게 관련 키워드 포함
- [ ] **간결성**: 125자 이내

```markdown
❌ 나쁜 예: ![image](screenshot.png)
✅ 좋은 예: ![Next.js App Router 폴더 구조 스크린샷](screenshot.png)
```

### 4. 태그 선택 - 중요도: 🟡 중간

**SEO 영향**: 콘텐츠 분류, 내부 링크 구조

- [ ] **관련성**: 게시글 내용과 직접 관련된 태그만 선택
- [ ] **개수**: 3-5개 정도가 적당
- [ ] **일관성**: 기존 태그 재사용 (새 태그 남발 방지)
- [ ] **구체성**: 너무 일반적인 태그 지양

```
❌ 나쁜 예: #개발 #코딩 #프로그래밍 #IT #기술
✅ 좋은 예: #Next.js #React #ServerComponents #성능최적화
```

### 5. 썸네일 이미지 (선택사항) - 중요도: 🟢 낮음

**SEO 영향**: 소셜 미디어 공유 시 클릭률

- [ ] **크기**: 1200x630px (Open Graph 권장 크기)
- [ ] **파일 크기**: 1MB 이하
- [ ] **내용**: 게시글 주제를 시각적으로 표현
- [ ] **텍스트**: 제목이나 핵심 메시지 포함 (선택)

**참고**: 썸네일을 업로드하지 않으면 기본 이미지 또는 동적 생성 이미지가 사용됩니다.

### 6. 콘텐츠 구조 - 중요도: 🔴 높음

**SEO 영향**: 가독성, 검색 엔진 이해도

- [ ] **제목 계층**: H1(제목) → H2(섹션) → H3(하위섹션) 순서 유지
- [ ] **단락 구분**: 3-4문장마다 단락 나누기
- [ ] **리스트 활용**: 항목 나열 시 불릿 포인트 사용
- [ ] **코드 블록**: 코드는 반드시 코드 블록으로 감싸기
- [ ] **내부 링크**: 관련 게시글 링크 추가

```markdown
# 메인 제목 (자동 생성됨)

## 소개

내용...

## 주요 내용

### 세부 항목 1

내용...

### 세부 항목 2

내용...

## 결론

내용...
```

### 7. 키워드 밀도 - 중요도: 🟡 중간

**SEO 영향**: 검색 순위

- [ ] **자연스러움**: 키워드를 자연스럽게 사용
- [ ] **밀도**: 전체 단어의 1-2% 정도
- [ ] **위치**: 제목, 첫 단락, 소제목에 포함
- [ ] **과도 사용 금지**: 키워드 스터핑 방지

## 🤖 자동화된 항목 (신경 쓰지 않아도 됨)

### 자동 생성되는 메타데이터

```typescript
// 시스템이 자동으로 생성
{
  title: "게시글 제목 | ZXCVB Blog",
  description: "게시글 요약",
  openGraph: {
    type: "article",
    publishedTime: "2025-11-17T...",
    authors: ["작성자 이름"],
  },
  // ... 기타 메타데이터
}
```

### 자동 생성되는 구조화된 데이터

```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "게시글 제목",
  "author": { "@type": "Person", "name": "작성자" },
  "datePublished": "2025-11-17"
  // ... 자동 생성
}
```

### 자동 업데이트되는 Sitemap

```xml
<!-- 빌드/재검증 시 자동 추가 -->
<url>
  <loc>https://zxcvb-blog.com/posts/123</loc>
  <lastmod>2025-11-17</lastmod>
  <changefreq>weekly</changefreq>
  <priority>0.8</priority>
</url>
```

## 📊 게시글 작성 후 확인사항

### 즉시 확인 (로컬/스테이징)

- [ ] 제목과 요약이 적절한지 확인
- [ ] 모든 이미지에 alt 텍스트가 있는지 확인
- [ ] 제목 계층 구조가 올바른지 확인
- [ ] 내부 링크가 정상 작동하는지 확인

### 배포 후 확인 (프로덕션)

- [ ] Google Search Console에서 인덱싱 요청
- [ ] 소셜 미디어 공유 미리보기 확인
  - [Facebook Sharing Debugger](https://developers.facebook.com/tools/debug/)
  - [Twitter Card Validator](https://cards-dev.twitter.com/validator)
- [ ] 모바일에서 가독성 확인
- [ ] 페이지 로딩 속도 확인 (Lighthouse)

### 1주일 후 확인

- [ ] Google Search Console에서 검색 노출 확인
- [ ] 클릭률(CTR) 확인
- [ ] 필요시 제목/요약 수정

## 🎯 SEO 점수 자가 진단

각 항목을 체크하고 점수를 계산하세요.

### 필수 항목 (각 20점)

- [ ] 제목이 50-60자 이내이고 키워드 포함 (20점)
- [ ] 요약이 120-160자 이내이고 명확함 (20점)
- [ ] 모든 이미지에 alt 텍스트 있음 (20점)
- [ ] 제목 계층 구조가 올바름 (20점)
- [ ] 3-5개의 관련 태그 선택 (20점)

**총점**: \_\_\_/100점

- **90-100점**: 완벽한 SEO 최적화 ✅
- **70-89점**: 양호, 일부 개선 필요 ⚠️
- **70점 미만**: 개선 필요 ❌

## 💡 팁

### 제목 작성 팁

1. 숫자 사용: "5가지 방법", "10분 만에"
2. 질문 형식: "어떻게 ~할까?"
3. 해결책 제시: "~하는 방법", "~완벽 가이드"
4. 긴급성: "2025년 최신", "지금 바로"

### 요약 작성 팁

1. 첫 문장에 핵심 키워드
2. 독자가 얻을 수 있는 가치 명시
3. 구체적인 숫자나 결과 포함
4. 행동 유도 문구 추가

### 콘텐츠 작성 팁

1. 첫 단락에 핵심 내용 요약
2. 짧은 문장과 단락 사용
3. 시각 자료 적극 활용
4. 실제 예시와 코드 포함
5. 결론에서 핵심 내용 재강조
